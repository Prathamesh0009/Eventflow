## 1. Purpose

## 3. Frontend Stack

### 3.1 Next.js (React + TypeScript)

**Responsibilities:**

* Web dashboard
* Event visualization
* Admin & producer management

**Why Next.js:**

* Industry standard in Germany
* Server-side rendering (SSR) & API routes
* Strong ecosystem & hiring demand
* Works well with cloud deployments

**Why TypeScript:**

* Type safety
* Better collaboration
* Strong signal of production readiness

---

## 4. Backend Stack

### 4.1 Node.js

**Why Node.js:**

* Non-blocking I/O (good for event ingestion)
* Widely used in startups and SMEs
* Strong ecosystem

### 4.2 NestJS (Framework)

**Why NestJS:**

* Opinionated architecture (controllers, services, modules)
* Excellent for scalable backend systems
* Inspired by Spring Boot (very familiar to German companies)
* Built-in support for:

  * Validation
  * Dependency Injection
  * Testing

---

## 5. Authentication & Security

### 5.1 JWT-Based Authentication

**Approach:**

* Access tokens for API calls
* Refresh tokens for session renewal

**Why JWT:**

* Stateless
* Cloud-friendly
* Simple for MVP

**Future extensions:**

* OAuth2
* SSO
* Role-based access control

---

## 6. Data Layer

### 6.1 PostgreSQL (Event Store)

**Why PostgreSQL:**

* ACID-compliant
* Excellent JSON support
* Familiar to recruiters
* Works well with analytics later

**Event Storage Strategy:**

* Append-only table
* Indexed by:

  * Event type
  * Timestamp
  * Source

---

### 6.2 Redis (Cache / Queue)

**Why Redis:**

* Lightweight message buffering
* Fast
* Simple to reason about

**Usage in MVP:**

* Temporary event queue
* Retry handling
* Rate limiting

---
